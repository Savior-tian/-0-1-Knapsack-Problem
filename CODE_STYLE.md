# Python 编程规范（本项目执行版）

以下规范在本次作业中统一执行。

## 1) 语句缩进

- 使用 4 个空格缩进。
- 禁止使用 Tab 缩进。

## 2) 变量命名

- 变量、函数、方法：`snake_case`。
- 布尔变量建议使用语义前缀，如 `is_`、`has_`、`can_`。
- 禁止使用单字符无语义变量名（循环计数器 `i/j/k` 除外）。

## 3) 每行最多字符数

- 最大行宽：88 字符。
- 长参数列表使用多行换行，不用反斜杠续行。

## 4) 函数最大行数

- 单个函数不超过 80 行（含空行与注释）。
- 超过上限时拆分为私有辅助函数。

## 5) 函数、类命名

- 类名：`PascalCase`。
- 函数/方法名：`snake_case`。
- 私有辅助函数使用前缀 `_`。

## 6) 常量

- 模块级常量使用 `UPPER_CASE`。
- 常量统一放在模块头部导入之后。

## 7) 空行规则

- 顶级函数/类之间保留 2 个空行。
- 类内方法之间保留 1 个空行。
- 逻辑分段可用 1 个空行提升可读性。

## 8) 注释规则

- 注释只保留“为什么这么做”，避免解释显而易见的“做了什么”。
- 优先使用简短 docstring 描述输入/输出与约束。
- 删除冗余和过时注释。

## 9) 操作符前后空格

- 二元操作符两侧必须有空格（如 `a + b`、`x == y`）。
- 关键字参数赋值不加空格（如 `func(a=1)`）。

## 10) 其他规则

- 导入分组并自动排序（标准库 / 第三方 / 本地模块）。
- 避免未使用导入、未使用变量。
- 不捕获过宽异常；异常信息需可定位。

## 执行与检查

- 使用 Ruff 执行 lint + 格式化。
- 命令：
  - `python -m ruff check .`
  - `python -m ruff format .`
- 函数行数使用脚本辅助检查，阈值为 80 行。
